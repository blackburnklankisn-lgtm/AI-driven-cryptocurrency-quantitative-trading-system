本次变更:
1. 将 Phase 2 外部数据源启动预采集从 LiveTrader 构造函数移到 _run_loop 启动后的异步 warmup 线程，避免 API 启动被 onchain 请求阻塞。
2. 修复前端构建脚本，避免 PowerShell 将 Node stderr 警告误判为构建失败，并去掉重复的 npm run build 调用。
3. 前端安装包与后端 backend_trader.exe 已按最新代码重新构建。

验证结果摘要:
1. 新版 backend_trader.exe 在纸交易模式下启动后，/api/v1/health 在 15.61 秒返回 200。
2. /api/v1/status 正常返回运行状态、权益与持仓摘要。
3. Electron 安装包已重新生成，最新文件与校验值已写入当前发布目录。

注意事项:
1. 当前 onchain 使用 public 数据源，无需 API KEY。
2. 首次启动期间外部数据 warmup 仍会在后台执行，但不再阻塞本地 HTTP API 对外提供服务。
3. 若后续切换到 glassnode 或 cryptoquant，再补充对应环境变量密钥。
